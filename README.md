# Linux 기본 프로세스 관리 명령어 정리

## 📌 소개
이 문서는 **top**, **ps**, **jobs**, **kill** 명령어에 대한 개념, 사용 예시, 옵션 등을 정리한 문서입니다.  


---

## 🖥️ 1. top — 실시간 프로세스 모니터링

### 🔹 개요
`top` 명령어는 **현재 실행 중인 프로세스와 시스템 상태(CPU, 메모리 등)** 를 실시간으로 보여주는 모니터링 도구입니다.

### 🔹 실행 예시
```
top
```

### 🔹 주요 화면 구성
![top example](https://github.com/user-attachments/assets/85384eae-e1da-478a-a086-dc56fb560737)

### 🔹 주요 기능 및 키 조작
- **P** : CPU 사용량 기준 정렬
- **M** : 메모리 사용량 기준 정렬
- **K** : 특정 PID 종료 신호 보내기
- **q** : 종료

### 🔹 장점
- 실시간 모니터링 가능
- 다양한 기준으로 정렬 가능
- 상호작용 키가 다양함

---

## 📄 2. ps — 현재 실행 중인 프로세스 출력

### 🔹 개요
`ps` 는 **스냅샷 형태로 현재 실행 중인 프로세스 목록** 을 출력하는 명령어입니다. top과 달리 실시간 업데이트는 되지 않습니다.

### 🔹 자주 사용하는 옵션
| 옵션 | 설명 |
|------|------|
| `ps aux` | 모든 사용자 프로세스 출력 |
| `ps -ef` | 풀 포맷으로 전체 프로세스 출력 |
| `ps -u 사용자` | 특정 사용자의 프로세스만 출력 |

### 🔹 예시
```
ps aux | grep python
```

### 🔹 주요 화면 구성
![top example](https://github.com/user-attachments/assets/2a2c0339-337f-4a22-b0bb-8ae2a436668e)

## 🧵 3. jobs — 백그라운드 작업 목록 보기

### 🔹 개요
`jobs` 명령어는 **현재 쉘에서 실행 중이거나 중지된 백그라운드 작업 목록** 을 보여줍니다.

### 🔹 예시
```
jobs
```

### 🔹 출력 예시
```
[1]+  Running   python script.py &
[2]-  Stopped   vim
```

### 🔹 관련 명령어
- **bg %1**: 백그라운드에서 계속 실행
- **fg %1**: 포그라운드로 가져오기

---

## 🔫 4. kill — 프로세스 종료 명령어

### 🔹 개요
`kill` 은 **특정 프로세스에 신호(signal)를 보내 종료하거나 특정 동작을 유도하는 명령어** 입니다.

### 🔹 자주 사용되는 신호(SIGNAL)
| 신호 번호 | 신호명 | 설명 |
|-----------|--------|------|
| 9 | SIGKILL | 강제 종료 |
| 15 | SIGTERM | 정상 종료 요청 (기본값) |
| 2 | SIGINT | 인터럽트 (Ctrl + C) |

### 🔹 예시
```
kill -9 1234
```

### 🔹 프로세스 찾기 + 종료 조합
```
ps aux | grep python
kill -15 PID
```
